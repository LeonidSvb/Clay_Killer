FANFIC GENERATOR BOT — СТАТУС ПРОЕКТА
=======================================
Дата: 01.04.2026


СТРУКТУРА ПРОЕКТА
-----------------
bot.py                  — точка входа, запуск, регистрация команд меню
states.py               — FSM состояния (квиз, глава, управление)
techniques.json         — 28 нарративных техник с метаданными
requirements.txt        — aiogram 3.x, httpx, python-dotenv
run.bat                 — запуск двойным кликом

handlers/
  start.py              — /start, welcome меню, квиз создания истории (4 вопроса)
  chapter.py            — /next, 8 кнопок после главы, двухуровневый выбор техники
  chapters.py           — /chapters, библиотека глав, навигация пред/след
  manage.py             — /list, /load, /regen, удаление с подтверждением названием

logic/
  generate.py           — генерация главы + подбор техник через LLM
  summarize.py          — summary + обновление персонажей после каждой главы
  state.py              — add_chapter, apply_summary, rollback_chapter
  storage.py            — JSON read/write, create/delete story

data/
  user_XXXXXXXXX.json   — история пользователя (локально, не в git)


КОМАНДЫ БОТА
------------
/start      — главное меню (продолжить / главы / мои истории / новая)
/next       — следующая глава
/chapters   — библиотека всех глав с навигацией ← →
/list       — карточки историй с кнопками [Читать] [Главы] [Удалить]
/load 2     — переключиться на историю номер 2
/regen      — откатить последнюю главу и выбрать как переписать


КАК ХРАНЯТСЯ ДАННЫЕ (JSON на пользователя)
-------------------------------------------
story {
  title, created_at
  state {
    genre, hero, world, start_situation
    characters []        — обновляется после каждой главы
    chapter_count
  }
  all_chapters []        — полный текст всех глав (добавлено с гл.7)
  last_chapters []       — последние 3 главы (контекст для LLM)
  summaries []           — краткое содержание последних 10 глав
  direction              — направление для следующей главы
}


СИСТЕМА ТЕХНИК (двухуровневая)
-------------------------------
После главы — 8 кнопок:
  [Продолжить] [Написать своё]
  [Замедлить/атмосфера] [Экшен/динамика] [Углубить персонажа]
  [Повернуть сюжет] [Раскрыть мир] [Отношения]

Нажал категорию → LLM выбирает 4-5 техник из techniques.json,
адаптирует под персонажей, возвращает список с описанием →
пользователь выбирает кнопкой 1/2/3...

techniques.json — 28 техник по 6 категориям:
  pacing_slow     — slow burn, внутренний монолог, передышка, сенсорное погружение
  pacing_fast     — экшен-вспышка, прорыв на грани, клиффхэнгер, обратный отсчёт
  character       — маска трескается, точка разлома, флэшбек, холодный расчёт
  plot            — триумф над противником, раскрытие силы, предательство, ложный след...
  world           — погружение в мир, система силы, фракции, ритуал, древняя тайна
  relationships   — медленный огонь, клятва верности, уважение врага, передача мудрости...


МОДЕЛЬ
------
OpenRouter → deepseek/deepseek-chat
$0.32 input / $0.89 output за 1M токенов
163K контекст
Параметры генерации: temperature 0.8, max_tokens 3000


ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ
---------------------
- Главы 1-6 текущей истории не в all_chapters (добавили позже) — только summaries
- FSM хранится в памяти (MemoryStorage) — при рестарте бота сбрасывается
- Одна история на пользователя "активная" — остальные доступны через /list


ПЛАН НА БУДУЩЕЕ
---------------

ПРИОРИТЕТ 1 — Exa AI real-time research
  Что: перед генерацией главы LLM делает 2-3 поисковых запроса через Exa,
       получает реальные данные (названия мест, новости, детали) и вставляет
       в контекст генерации
  Зачем: реалистичность — реальные названия кафе, пляжей, новости в реальном времени
  Как: опциональная галочка в меню техник или отдельная кнопка "добавить реализм"
  Ключ: OPENROUTER_API_KEY уже есть, нужен EXA_API_KEY (у пользователя есть)
  Файлы: logic/research.py (новый), logic/generate.py (добавить вызов до генерации)

ПРИОРИТЕТ 2 — Структурированные арки
  Что: файл story_arcs.json с шаблонами для жанров (культиваторка, фэнтези и тд)
       Каждая арка задаёт: какие главы что должны раскрывать, какие техники уместны,
       какие события должны произойти к определённой главе
  Зачем: проверенная структура лучших произведений жанра вместо рандома
  Как: два режима — "свободный" (как сейчас) и "по структуре" (следует арке)
  Файлы: story_arcs.json (новый), logic/arc_manager.py (новый)

ПРИОРИТЕТ 3 — Whisper транскрипция голосовых
  Что: бот принимает голосовые сообщения, отправляет в OpenAI Whisper,
       получает текст и обрабатывает как обычное сообщение
  Зачем: пользователь хочет диктовать ответы в квизе и направления
  Стоимость: ~$0.006 за минуту аудио

ПРИОРИТЕТ 4 — Компрессия памяти (global_summary)
  Что: когда summaries > 10 глав, старые сжимаются в global_summary через LLM
  Зачем: при 50+ главах контекст начнёт деградировать
  Пока: текущая схема (10 recent summaries) покрывает ~50 глав без проблем

ПРИОРИТЕТ 5 — Экспорт истории
  Что: команда /export — отправляет все главы одним текстовым файлом
  Зачем: читать удобнее в читалке, сохранить результат
