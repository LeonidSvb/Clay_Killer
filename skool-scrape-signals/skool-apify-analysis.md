## 1 пост = 1 result, или пост + комментарии = N results?

**1 пост = 1 result ($0.0045), комментарии — БЕСПЛАТНО.**

Каждый пост — это один dataset item. Все комментарии (сколько бы их ни было — 1 или 200) вложены внутрь этого объекта как массив. Если включить `includeComments: true`, получаешь пост + весь тред комментов за те же $0.0045. Это очень выгодно.

## Безопасность

**Использование куков своего аккаунта:**
- Относительно безопасно — куки это просто сессия браузера
- Риск: Skool может забанить сессию (не аккаунт), если обнаружит скрейпинг
- Практика: использовать cookies от дополнительного/throwaway аккаунта Skool, **не основного**
- Если основной аккаунт заблокируют — потеряешь доступ к комьюнити

**Без куков:**
- Публичный discovery (`/discovery?q=...`) — работает без куков (поиск групп)
- Посты и члены закрытых комьюнити — **куки обязательны**, нужно быть членом группы

## Фильтрация по комьюнити

Да, полная поддержка. В `startUrls` передаешь конкретный URL:
```json
{
  "startUrls": [{"url": "https://www.skool.com/ai-automation-mastery"}],
  "tab": "community",
  "includeComments": true,
  "maxItems": 50
}
```
Можно передать несколько URL сразу — несколько комьюнити в одном запуске.

## Оптимизация расходов

### Стратегия для поиска лидов (automation/outreach запросы)

**Сценарий:** 3 комьюнити, 50 новых постов/день каждое = 150 постов/день

| | Расчет |
|---|---|
| Постов в день | 150 |
| Цена за пост | $0.0045 |
| Стоимость постов | $0.675/день |
| Старт (1 GB × 1 run) | $0.005 one-time |
| **Итого в месяц** | **~$20.25** |

### FREE аккаунты (у тебя 4 штуки)
Каждый FREE аккаунт: ~$5/месяц бесплатного кредита

| Аккаунт | Free кредит |
|---|---|
| standartdevelop | $5 |
| quarrelsome_rattlesnake | $5 |
| fascinated_seal | $5 |
| grand_joy | $5 |
| **Итого** | **$20/месяц** |

**150 постов/день × 30 дней = 4,500 постов = $20.25** — почти укладывается в 4 FREE аккаунта.

### Как снизить расход до минимума

1. **Ограничивать `maxItems`** — не скрейпить все 1000+ постов, только последние 30-50
2. **Скрейпить только нужные комьюнити** — не все группы, а только те где реально есть клиенты
3. **Мониторинг-режим** — Members scraper умеет возвращать только НОВЫЕ записи с прошлого запуска (экономит >80% повторных запросов)
4. **Расписание** — запускать 1 раз в день утром, не чаще
5. **Без `includeComments`** для первичного скрейпа — собрать посты, потом догрузить комменты только по нужным

### Пример дешевого рана
```json
{
  "startUrls": [{"url": "https://www.skool.com/ai-automation-mastery"}],
  "tab": "community",
  "includeComments": false,
  "maxItems": 30
}
```
**Стоимость: 30 × $0.0045 + $0.005 = $0.14 за запуск**

Потом фильтруешь посты по ключевым словам (automation, outreach, help, need, looking for) и берешь комменты только к нужным.

## Workflow для поиска лидов

1. Skool посты → Apify (posts scraper, последние 30-50 постов, без комментов)
2. Фильтр по ключевым словам через Claude/GPT (automation, outreach, need help, looking for)
3. Если пост релевантный → догружаем комменты к нему (опционально)
4. Находим профиль автора в посте (user.firstName, user.lastName + ссылка на LinkedIn в metadata)
5. Идем в LinkedIn → пишем

**Стоимость сигнала:** ~$0.01-0.05 на найденного лида (дешевле большинства lead gen инструментов)
