## Deep mode — архитектура (требует исследования)

Текущий подход: все страницы Exa → один большой текст → один LLM вызов → summary + scoring.
Проблема: при 5 subpages текст огромный, часть информации теряется в summary.

Три варианта для исследования:

- [ ] **Вариант A — Page-by-page facts extraction (двухуровневый)**
      Каждая страница → LLM извлекает только сухие факты (что продают, кому, где, размер)
      Финально → LLM объединяет и дедуплицирует факты в одну запись
      Плюс: каждая страница обрабатывается независимо, можно параллелить
      Минус: N+1 LLM вызовов на компанию, дороже, сложнее

- [ ] **Вариант B — Весь текст в контекст (текущий, улучшенный)**
      Убрать max_llm_chars cap + убрать max_tokens лимит
      GPT-OSS-120b контекст 128k → 75k chars входит легко
      Улучшить промпт: явно указать "extract from ALL pages, don't miss subpages"
      Плюс: один LLM вызов, просто
      Проверить: был ли confidence drop из-за обрезки текста или из-за max_tokens=1500?
      (скорее второе — JSON обрезался, не текст)

- [ ] **Вариант C — Per-page summary → финальный синтез**
      Каждая страница → LLM делает короткий summary (3-5 предложений, только факты)
      Все mini-summaries → финальный LLM вызов синтезирует в полную запись
      Плюс: ни одна страница не теряется, финальный контекст компактный
      Минус: N+1 вызовов, дороже варианта B, дешевле варианта A

      Рекомендация для теста: сначала попробовать Вариант B (просто убрать лимиты).
      Если confidence всё ещё низкий после fix max_tokens → переходить к Варианту C.
