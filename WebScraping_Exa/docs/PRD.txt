1. ЧТО МЫ СТРОИМ
----------------------------------------------------------------

Система быстрого понимания компаний для принятия решений в outreach.

Не скрейпер. Не база данных. Не мониторинг.

Это pipeline: подаёшь список URL → получаешь обогащённый CSV
с summary, ICP оценкой и любыми другими переменными которые нужны.

Входные данные:  CSV с колонкой Company Website
Выходные данные: тот же CSV + новые колонки из LLM

Интерфейсы: CLI (pipeline.py) + Streamlit UI (streamlit_app.py)
Деплой: локально сейчас, потом на свой сервер

3. PIPELINE — КАК РАБОТАЕТ
----------------------------------------------------------------

PASS 0 — Pre-screen (бесплатно, ~60 сек на 5000 URL)
  Для каждого URL: async HTTP GET, первые 10KB HTML
  Классификация:
    html_light  → есть реальный текст в body, нет JS-фреймворков
    js_heavy    → React/Vue/Next/Angular (пустой body или SPA сигнатуры)
    blocked     → 403/503/Cloudflare
    dead        → timeout/DNS fail → SKIP

PASS 1a — Custom scraper (для html_light, бесплатно)
  BeautifulSoup: убрать nav/footer/script/style
  Получить чистый текст страницы
  Если текст < 500 chars → переклассифицировать в js_heavy

PASS 1b — Exa API (для js_heavy + blocked)
  maxAgeHours: 24 (кэш + livecrawl fallback)
  concurrency: 50
  text: true (не summary — текст отдаём своему LLM)
  subpages: нет на первом проходе

PASS 2 — LLM extraction (для всех живых сайтов)
  Модель: openai/gpt-oss-120b
  Provider sort: "throughput" (ОБЯЗАТЕЛЬНО — иначе 7x медленнее)
  concurrency: 50
  Input: system_context.txt + выбранный промпт + текст сайта
  Output: JSON с произвольными полями (определяется промптом)
  Всегда включает поле: "confidence": 0-10

PASS 3 — Retry для слабых результатов (confidence < 6)
  Только для ~15-20% лидов
  Exa subpages: 5 страниц (about/services/solutions/clients/industries)
  Тот же LLM вызов с большим контекстом
  Перезаписывает результат Pass 2

ФИНАЛ — Merge результатов в DataFrame
  Выбранные колонки → CSV


----------------------------------------------------------------
4. LLM — ПРОМПТЫ И ГИБКОСТЬ
----------------------------------------------------------------

СИСТЕМА ПРОМПТОВ:

  system_context.txt
    Общий контекст который добавляется к КАЖДОМУ промпту.
    Здесь: описание своей компании, ICP критерии, правила вывода.
    Пример содержания:
      "You are analysing companies for outreach on behalf of
       System Hustle, a B2B recruitment company.
       ICP: logistics companies in Canada/USA, 50-500 employees,
       B2B focused, hiring actively.
       Always output valid JSON only."

  company_profile.txt
    Большой подробный summary: что делают, кому продают,
    geography, services, differentiator, доказательства.
    Output: { "summary": string, "confidence": 0-10 }

  icp_score.txt
    Focused на ICP решение.
    Output: {
      "icp_fit": 0-10,
      "icp_reasons": string,
      "b2b": boolean,
      "geography": string,
      "company_size_estimate": string,
      "confidence": 0-10
    }

  Можно комбинировать: company_profile.txt + icp_score.txt
  в одном вызове (один промпт включает оба).

МЕНЕДЖЕР КОНФИГУРАЦИЙ:
  Пользователь может сохранить комбинацию настроек:
    - какой промпт
    - какие колонки сохранять на выход
    - system_context
  Сохраняется как JSON в prompts/saved_configs/
  В Streamlit: dropdown выбор сохранённой конфигурации

ПАРСИНГ JSON ИЗ LLM:
  LLM иногда оборачивает в ```json ... ``` — нужен robust парсер.
  Если JSON не распарсился → поле "raw" с сырым текстом.
  Данные не теряются.


----------------------------------------------------------------
5. ЭКОНОМИКА
----------------------------------------------------------------

Бенчмарк (реальные тесты):
  Exa: 7.6 лидов/сек при concurrency=50, maxAgeHours=24
  LLM: ~7 лидов/сек при concurrency=50 (оценка для длинных промптов)
  Pre-screen: 5000 URL за ~60 сек (бесплатно)

Узкое место: Exa (~3-5/сек с учётом Pass 3 retries)

Скорость:
  5000 лидов  → ~15-20 минут
  30000 лидов → ~90-120 минут

Стоимость на 5000 лидов (с pre-screen оптимизацией):
  html_light (50%): бесплатный scraper         = $0
  js_heavy   (35%): Exa 1750 pages × $0.001    = $1.75
  blocked    (15%): Exa 750 pages × $0.001     = $0.75
  Pass 3 retry (~15%): 750 × 5 pages × $0.001 = $3.75
  LLM (gpt-oss-120b): ~4750 × $0.0002         = $0.95
                                         Total ≈ $7.20

Стоимость на 30000 лидов:
  Примерно $43/месяц

6. STREAMLIT UI — ФУНКЦИОНАЛЬНОСТЬ
----------------------------------------------------------------

UPLOAD TAB:
  - Загрузка CSV
  - Авто-детект колонки с URL (по имени + контенту)
  - Показ метрик: всего строк / уже обработаны / к обработке
  - Выбор лимита: пресеты 50/100/500/1000/All + Custom
  - Чекбокс "Skip rows that already have summary"

CONFIGURATION TAB (сайдбар или отдельная вкладка):
  - Выбор промпта из dropdown (файлы из /prompts/)
  - Редактирование system_context в text_area
  - Редактирование промпта в text_area (с возможностью сохранить)
  - Сохранить конфигурацию / Удалить сохранённую
  - API ключи (Exa, OpenRouter)
  - Настройки: concurrency, confidence threshold для Pass 3

COLUMNS TAB (после получения первых результатов):
  - Показывает все поля из JSON которые вернул LLM
  - Multiselect: какие колонки добавить в итоговый CSV
  - Preview: название колонки → пример значения
  - st.data_editor для просмотра и ручного редактирования строк

PROGRESS (во время работы):
  - Progress bar
  - live статус: N/total | X.X leads/sec | ETA MM:SS | Pass 2: N | Pass 3: N
  - Таблица последних 15 результатов (url + summary превью + icp_fit + ok/err)
  - Кнопка Stop

RESULTS:
  - 4 метрики: Processed / Success % / Time / Cost ($)
  - Preview таблица (st.data_editor — можно редактировать)
  - Download enriched CSV (полный)
  - Download results only (только обработанные строки)
  - Failed rows expander (что упало и почему)
  - Кнопка "Process more" (не сбрасывает загруженный файл)

7. УПРАВЛЕНИЕ КОЛОНКАМИ
----------------------------------------------------------------

Проблема: у пользователя уже есть "Website Summary" в CSV.
Новый pipeline может генерировать другой summary.

Решение: пользователь сам называет выходную колонку.
  - По умолчанию: берётся из промпта (напр. "summary")
  - Можно переименовать перед сохранением
  - Если колонка с таким именем уже есть → предупреждение + выбор:
    overwrite / add new (с суффиксом _v2) / cancel

Все остальные колонки оригинального CSV сохраняются as-is.

9. ПРЕСЕТЫ ПРОМПТОВ НА СТАРТ
----------------------------------------------------------------

1. company_profile.txt
   Input:  текст сайта
   Output: { summary: string (800-1200 chars), confidence: 0-10 }
   Цель:   source of truth для дальнейших enrichments

2. icp_score.txt
   Input:  текст сайта
   Output: { icp_fit: 0-10, icp_reasons: string,
             b2b: boolean, geography: string,
             company_size_estimate: string, confidence: 0-10 }
   Цель:   быстрое ICP решение без длинного summary

3. company_full.txt
   Input:  текст сайта
   Output: { summary: string, icp_fit: 0-10, icp_reasons: string,
             b2b: boolean, geography: string, services: array,
             target_market: string, confidence: 0-10 }
   Цель:   полный профиль за один вызов

Добавление новых промптов: просто создать новый .txt файл в /prompts/
Streamlit подхватит автоматически (читает папку при загрузке).


----------------------------------------------------------------
10. ЧТО НЕ ВХОДИТ В ЭТОТ ПРОЕКТ
----------------------------------------------------------------

- Скрапинг контактов (email, phone) → отдельный проект
- Скрапинг postal codes → отдельный проект
- Google Sheets direct write-back → CSV only, импорт руками
- Мониторинг изменений сайта во времени → не нужно
- Multi-user auth → один пользователь
