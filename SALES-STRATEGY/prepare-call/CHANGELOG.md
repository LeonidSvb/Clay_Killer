# Changelog
Только структурные изменения системы (шаблоны, конвенции, инструменты) — не рутинная работа по прospectам.

## 2026-07-04

- **brief.md переработан в сквозной формат** (6 разделов: кто он → бизнес → экономика ниши → ICP/сигналы → ретейнер → red flags/тон). Вопросы/питч/возражения/цена вынесены только в call-script.md, больше не дублируются.
- **Слиты в один файл** `prospect_research_todo.md` + `call-intel-template.md` → `templates/prospect-research-template.md` (процесс ресёрча + структура брифа с примерами в одном месте).
- **OPENING в call-script-template.md переписан:** убрана самопонижающая рамка "fit-check, wrap early if not relevant". Приоритет крючка: свежий рыночный сигнал > подтверждение ICP из переписки. Scarcity-реплика ("I'll be straight — you made the cut") — только реактивно, если прospect сам поднимает скепсис/время.
- **Exa API подключён** как основной инструмент ресёрча вместо обычного WebSearch — даёт более глубокий контент (реальные тексты страниц сайта, не только сниппеты). Ключ в `.env`, вызов через `scripts/utils/exa-search.sh "query" [numResults] [maxChars]`.
- **templates/ реорганизован:** `proposals/` (3 файла сгруппированы), `_reference/` (архив — `discovery-call-framework-saad.md`, исходный референс, полностью впитан в call-script-template.md), snake_case → kebab-case для оставшихся файлов.
- **Новые prospects/[name]/ папки** — с этого момента именуются `YYYY-MM-DD-name` (дата первого звонка) для автоматической хронологической сортировки. Старые папки не переименованы.
- **.gitignore добавлен** — `.env` не был в git, но защиты не было.
